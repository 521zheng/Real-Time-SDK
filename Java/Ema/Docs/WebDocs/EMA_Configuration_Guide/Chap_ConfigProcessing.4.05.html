Example: Default Server Behavior (IProvider)
The default (hard-coded) behavior for a Server adheres to the following configuration:
•
ServerType value=”RSSL_SOCKET”
•
CompressionType value=”None”
•
TcpNoDelay value=”1”
•
Port value= “14002”
Note that unlike EMA’s default behavior of choosing the first IProvider component in the IProviderList, EMA applications will not choose the first or Server in their respective lists. Instead, if an application wants to use a specific server or dictionary configuration, the application must explicitly configure it in the appropriate IProvider section of the XML file.
