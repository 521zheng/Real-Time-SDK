9.2.1
Market Maker Request Message
A Market Maker request message is encoded using ReqMsg and sent by OMM consumer applications. The request specifies the name of an item in which the consumer is interested.
To receive updates, a consumer can make a “streaming” request by setting the ReqMsg.InterestAfterRefresh to true. If the flag is not set, the consumer requests a “snapshot,” and the final part of the refresh indicates all responses have been received for the snapshot. Updates may be received in either case if the refresh has multiple parts.
To stop updates, a consumer can pause an item (if the provider supports this functionality). For more information, refer to the Message API Java Edition Developers Guide.
