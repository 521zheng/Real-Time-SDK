Transport API Java Edition

Performance Tools
 
Example
Description
upajConsPerf
upajConsPerf measures the performance of the Transport API in consuming Level I Market Price content directly from an OMM provider or through TREP components (i.e., ADS, ADH, etc.).
To measure latency, the provider inserts a timestamp randomly in each burst of updates. The consumer then decodes the timestamp from the update to determine end-to-end latency. upajConsPerf also supports the measurement of posting latency.
upajNIProvPerf
upajNIProvPerf measures performance of the Transport API in providing Level I Market Price content to the TREP’s Advanced Data Hub (ADH).
The provider creates two categories of threads:
•
A main thread which collects and records statistical information.
•
Provider threads, each connecting to an ADH and providing market data.
upajProvPerf
upajProvPerf measures the performance of the Transport API in providing Level I Market Price content either directly to an OMM Consumer or to a TREP component (i.e., ADS or ADH).
You can configure upajProvPerf to provide updates at various rates. To measure latency, a timestamp is randomly placed in each burst of updates. The consumer then decodes the timestamp from the update to determine end-to-end latency.
upajTransportPerf
upajTransportPerf measures the performance of the Transport API, using the different connection types with variable message sizes. You can use upajTransportPerf to measure performance on all transport types, including shared memory, reliable multicast, TCP/IP, and HTTP.
