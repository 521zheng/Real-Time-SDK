RSSL Reactor Structure
Transport API Reactor

The RSSL Reactor structure is used by the Transport API Reactor to manage RSSL Reactor Channel connections.

 RSSL Reactor Channel Event Structure
 

The RSSL Reactor Channel Event structure is used to alert the application that an event has occurred on an RsslReactorChannel, and allows the user to provide the callbacks for processing the event.

Enumeration Type Documentation

enum RsslDownloadDictionaryMode
See also:
RsslReactorOMMConsumerRole
Enumerator:
RSSL_RC_DICTIONARY_DOWNLOAD_NONE 

(0) Do not automatically reequest dictionary messages.

RSSL_RC_DICTIONARY_DOWNLOAD_FIRST_AVAILABLE 

(1) Reactor searches RsslRDMDirectoryMsgs for the RWFFld and RWFEnum dictionaries. Once found, it will request the dictionaries and close their streams once all necessary data is retrieved. This option is for use with an ADS.

enum RsslReactorCallbackRet
See also:
loginMsgCallback, directoryMsgCallback, dictionaryMsgCallback, defaultMsgCallback, channelEventCallback
Enumerator:
RSSL_RC_CRET_SUCCESS 

(0) The message or event has been handled.

RSSL_RC_CRET_FAILURE 

(-1) The message or event failed to be handled. Returning this code will cause the RsslReactor to shutdown.

RSSL_RC_CRET_RAISE 

(-2) Used from domain-specific callbacks such as loginMsgCallback. Causes the defaultMsgCallback function to be called with the same message.

enum RsslReactorChannelRoleType
See also:

Enumerator:
RSSL_RC_RT_INIT 

(0) Unknown role

RSSL_RC_RT_OMM_CONSUMER 

(1) Indicates the RsslReactorChannel represents the connection of an OMM Consumer.

RSSL_RC_RT_OMM_PROVIDER 

(2) Indicates the RsslReactorChannel represents the connection of an OMM Provider.

RSSL_RC_RT_OMM_NI_PROVIDER 

(3) Indicates the RsslReactorChannel represents the connection of an OMM Provider.
