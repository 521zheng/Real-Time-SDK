RSSL Reactor Channel Event Structure
RSSL Reactor Structure

The RSSL Reactor Channel Event structure is used to alert the application that an event has occurred on an RsslReactorChannel, and allows the user to provide the callbacks for processing the event.

Enumeration Type Documentation

enum RsslReactorChannelEventType
See also:
RsslReactorChannelEvent
Enumerator:
RSSL_RC_CET_MIN 

Minimum value expected - this enumeration is for range checking

RSSL_RC_CET_INIT 

Unknown event type.

RSSL_RC_CET_CHANNEL_UP 

Channel has successfully initialized and can be dispatched. If the application presented any messages for setting up the session in their RsslReactorChannelRole structure in rsslReactorConnect() or rsslReactorAccept(), they will now be sent.

RSSL_RC_CET_CHANNEL_READY 

Channel has sent and received all messages expected for setting up the session. Normal use(such as item requests) can now be done.

RSSL_RC_CET_CHANNEL_DOWN 

Channel has failed(e.g. the connection was lost or a ping timeout expired) and will no longer be processed. The application should call rsslReactorCloseChannel() to clean up the channel.

RSSL_RC_CET_FD_CHANGE 

The file descriptor representing this channel has changed. The new and old Socket ID can be found on the RsslReactorChannel.

RSSL_RC_CET_WARNING 

An event has occurred that did not result in channel failure, but may require attention by the application.

RSSL_RC_CET_CHANNEL_DOWN_RECONNECTING 
RSSL_RC_CET_CHANNEL_OPENED 

Channel was opened by the application and can be used (occurs when watchlist is enabled and only appears in the channelOpenCallback).
