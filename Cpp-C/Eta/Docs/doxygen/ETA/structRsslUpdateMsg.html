RsslUpdateMsg Struct Reference
RsslUpdateMsg Structure, Initializers, and Flag values

The RsslUpdateMsg is used by providers(both interactive and non-interactive) to convey changes to data associated with an item stream. More...

#include <rsslUpdateMsg.h>

Data Fields

RsslUInt16 conflationCount
 Number of updates conflated into this update. A value of 0 means that this update is not conflated. Presence is indicated by RsslUpdateFlags::RSSL_UPMF_HAS_CONF_INFO.
RsslUInt16 conflationTime
 Time period(in miliseconds) over which updates were conflated. Presence is indicated by RsslUpdateFlags::RSSL_UPMF_HAS_CONF_INFO.
RsslBuffer extendedHeader
 Extended header attributes, used for per-domain defined attributes that are not used when determining stream uniqueness. Presence is indicated by RsslUpdatetFlags::RSSL_UPMF_HAS_EXTENDED_HEADER.
RsslUInt16 flags
 Flag values used to indicate optional member presence and/or stream behavior. The available options are defined by values present in RsslUpdateFlags.
RsslMsgBase msgBase
 Common message header members (streamId, domainType, msgKey, etc.), msgKey is optional and typically not sent as streamId can be used to match updates to a specific item stream.
RsslBuffer permData
 Contains permissioning information for this update. Presence is indicated by RsslUpdateFlags::RSSL_UPMF_HAS_PERM_DATA.
RsslPostUserInfo postUserInfo
 Information used to identify the user that posted this update to an upstream provider. Presence is indicated by RsslUpdateFlags::RSSL_UPMF_HAS_POST_USER_INFO.
RsslUInt32 seqNum
 Sequence number intended to help with temporal ordering. Typically, this will be incremented with every message, but may have gaps depending on the sequencing algorithm being used. Presence is indicated by RsslUpdateFlags::RSSL_UPMF_HAS_SEQ_NUM.
RsslUInt8 updateType
 Indicates domain-specific information about the type of content contained in this update. See rsslRDM.h for domain-specific enumerations for usage with the Thomson Reuters Domain Models.
