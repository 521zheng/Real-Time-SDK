RSSL Channel Buffer Operations
RSSL Write Operations

The RSSL Channel Buffer functions are used to acquire and release buffers from the guaranteed output buffers associated with an active rsslChannel or the shared buffer pool associated with an rsslServer.

RSSL_API RsslBuffer* rsslGetBuffer ( RsslChannel *  chnl,
RsslUInt32  size,
RsslBool  packedBuffer,
RsslError *  error 
)

Typical use:
This is called when a buffer is needed to write data to. Generally, the user will populate the RsslBuffer structure and then pass it to the rsslWrite function.

Parameters:
chnlRSSL Channel who requests the buffer
sizeSize of the requested buffer
packedBufferSet to RSSL_TRUE if you plan on packing multiple messages into the same buffer
errorRSSL Error, to be populated in event of an error
Returns:
RsslBuffer RSSL buffer to be filled in with valid memory

RSSL_API RsslBuffer* rsslPackBuffer ( RsslChannel *  chnl,
RsslBuffer *  buffer,
RsslError *  error 
)

Typical use:
This is called when the application wants to perform message packing. Call rsslGetBuffer to get a new buffer Call necessary encode functions Call rsslPackBuffer to write size of packed buffer and prepare for next packed buffer Repeat above two steps until ready to write Call rsslWrite with buffer

Parameters:
chnlRsslChannel that the message is being sent on.
bufferRssl buffer to be packed (length must be set to encoded size)
errorRSSL Error, to be populated in event of an error
Returns:
RsslBuffer Buffer pointing to new position to encode data to
RSSL_API RsslRet rsslReleaseBuffer ( RsslBuffer *  buffer,
RsslError *  error 
)

Typical use:
This is called when a buffer is done being used. The rsslWrite function will release the buffer if it successfully writes. The user should only need to use this function when they get a buffer that they do not need or when rsslWrite fails.

Parameters:
bufferRSSL buffer to be released
errorRSSL Error, to be populated in event of an error
Returns:
RsslRet RSSL return value
