Transport API Message Key
Transport API Message Structures

The RsslMsgKey structure contains attributes that are used to identify the contents of a particular stream. This information, as well as the domain type and Quality of Service information can be used to uniquely identify a stream. More...

Detailed Description

The RsslMsgKey is used in RsslRequestMsg to specify the item stream that the consumer wishes to open. After the provider has processed the request, any subsequent RsslResponseMsg should contain the same Stream ID and RsslMsgKey information that the consumer specified. Updates typically will not contain an RsslMsgKey unless the consumer specifies it in the initial RsslRequestMsg.

To use RsslMsgKey functionality include "rtr/rsslMsgKey.h"

(0x0010) This RsslMsgKey has a numeric identifier, contained in RsslMsgKey::identifier

RSSL_MKF_HAS_ATTRIB 

(0x0020) This RsslMsgKey has additional attribute information, contained in RsslMsgKey::encAttrib. The container type of the attribute information is contained in RsslMsgKey::attribContainerType.
