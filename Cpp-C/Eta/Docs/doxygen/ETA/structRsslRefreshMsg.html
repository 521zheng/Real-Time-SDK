RsslRefreshMsg Struct Reference
RsslRefreshMsg Structure, Initializers, and Flag values

The RsslRefreshMsg is provided as an initial response or when an upstream source requires a data resynchronization point. More...

#include <rsslRefreshMsg.h>

Data Fields

RsslBuffer extendedHeader
 Extended header attributes, used for per-domain defined attributes that are not used when determining stream uniqueness. Presence is indicated by RsslRefreshFlags::RSSL_RFMF_HAS_EXTENDED_HEADER.
RsslUInt16 flags
 Flag values used to indicate optional member presence and/or stream behavior. The available options are defined by values present in RsslRefreshFlags.
RsslBuffer groupId
 Group identifier. This contains information about the item group this stream belongs to. This typically contains a sequence of two-byte integers, where system components can append additional values to ensure uniqueness.
RsslMsgBase msgBase
 Common message header members (streamId, domainType, msgKey, etc.). The msgKey is optional and typically not sent as StreamId is used to determine the specific item stream.
RsslUInt16 partNum
 Part number is a 15-bit value typically used with multi-part refresh messages, value should start with 0 to indicate the initial part, each subsequent part in a multi-part refresh message should increment the previous partNum by 1. Presence is indicated by RsslRefreshFlags::RSSL_RFMF_HAS_PART_NUM.
RsslBuffer permData
 Contains permission authorization information for all content provided on this stream. Presence is indicated by RsslRefreshFlags::RSSL_RFMF_HAS_PERM_DATA.
RsslPostUserInfo postUserInfo
 Information used to identify the user that posted this update to an upstream provider. Presence is indicated by RsslRefreshFlags::RSSL_RFMF_HAS_POST_USER_INFO.
RsslQos qos
 Quality of Service information for the current item stream. See RsslQos for more details. Presence is indicated by RsslRefreshFlags::RSSL_RFMF_HAS_QOS.
RsslMsgKey reqMsgKey
 Optional Key providing unique identifier information for an item stream. If present, this contains the original request message key and is used to uniquely identify a stream.
RsslUInt32 seqNum
 Sequence number intended to help with temporal ordering. Typically, this will be incremented with every message, but may have gaps depending on the sequencing algorithm being used. Presence is indicated by RsslRefreshFlags::RSSL_RFMF_HAS_SEQ_NUM.
RsslState state
 Contains state information for the current item stream. See RsslState for more details.
