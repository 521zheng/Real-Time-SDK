struct  RsslFieldList
 The RsslFieldList container type allows the user to represent a collection of field identifier - value pairs, where the value can be a container type or a primitive type. The field identifier (RsslFieldEntry::fieldId) can be cross referenced with a field dictionary to determine the specific type of the content. The dictionary contains specific name and type information (e.g. RDMFieldDictionary). More...

Enumeration Type Documentation

enum RsslFieldListFlags
See also:
RsslFieldList
Enumerator:
RSSL_FLF_NONE 

(0x00) No RsslFieldList flags are present

RSSL_FLF_HAS_FIELD_LIST_INFO 

(0x01) The RsslFieldList has an RsslFieldList::dictionaryId and RsslFieldList::fieldListNum. If not present, RsslFieldList::dictionaryId is assumed to be 1.

RSSL_FLF_HAS_SET_DATA 

(0x02) The RsslFieldList payload contains set defined content. To decode, a set definition database is required - otherwise set data will be skipped while decoding.

RSSL_FLF_HAS_SET_ID 

(0x04) The RsslFieldList contains has an RsslFieldList::setId. This indicates which set definition is required to decode any set defined content. If not present but set data is included, this is assumed to be 0.

RSSL_FLF_HAS_STANDARD_DATA 

(0x08) The RsslFieldList contains standard encoded content (e.g. not set defined).
