Transport API Message Structures
Transport API Wire Format Message Package

The Transport API Message Package's message structures are required to encode or decode OMM messages.

Modules

 Transport API Message Base
 

The RsslMsgBase contains information that is shared across all Transport API Message structures.


 Transport API Message Key
 

The RsslMsgKey structure contains attributes that are used to identify the contents of a particular stream. This information, as well as the domain type and Quality of Service information can be used to uniquely identify a stream.


 RsslRequestMsg Reference Group
 

The Request Message allows a user to open a stream for the indicated content, or to change requested attributes/properties on an already open stream.


 RsslRefreshMsg Reference Group
 

The Refresh Message is used to provide an initial or subsequent data synchronization point, including quality of service and payload content. When used to respond to an RsslRequestMsg, a solicited RsslRefreshMsg can be used. If pushing out a content change to downstream applications, an unsolicited RsslRefreshMsg can be used. When the stream is instantiated because of an RsslRequestMsg, the streamId from the request should be used; when a provider instantiates the stream with an RsslRefreshMsg (e.g. Non-Interactive provider), a negative streamId should be used.


 RsslStatusMsg Reference Group
 

The Status Message is used to convey state information, permission change, or group Id change information on a stream. An interactive or non-interactive providing application can also close a stream with a status message.


 RsslUpdateMsg Reference Group
 

The Update Message is used to provide content change on an open stream.


 RsslCloseMsg Reference Group
 

The Close Message is used to indicate that a consuming application is no longer interested in content on the stream being closed.


 RsslPostMsg Reference Group
 

The Post Message is used to contribute content from a consumer into the platform. The post message payload can contain another OMM message, OMM container, or other opaque content.


 RsslAckMsg Reference Group
 

The Ack Message is used to acknowledge success/failure or completion of an RsslPostMsg or RsslCloseMsg.


 RsslGenericMsg Reference Group
 

The Generic Message allows applications to bidirectionally message without any implied interaction semantics (e.g. no Request/Response paradigm).


 RsslMsg Union Reference Group
 

OMM Message Union contains all messages. This includes constructs, functions, and values.
