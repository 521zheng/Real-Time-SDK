Enumeration Type Documentation

enum RsslUpdateFlags
See also:
RsslUpdateMsg, RsslMsg
Enumerator:
RSSL_UPMF_NONE 

(0x000) No RsslUpdateMsg flags are present

RSSL_UPMF_HAS_EXTENDED_HEADER 

(0x001) This RsslUpdateMsg has an extended header buffer, contained in RsslUpdateMsg::extendedHeader.

RSSL_UPMF_HAS_PERM_DATA 

(0x002) This RsslUpdateMsg has permission information, contained in RsslUpdateMsg::permData.

RSSL_UPMF_HAS_MSG_KEY 

(0x008) This RsslUpdateMsg has an RsslMsgKey included in the update, contained in RsslUpdateMsg::msgBase::msgKey

RSSL_UPMF_HAS_SEQ_NUM 

(0x010) This RsslUpdateMsg contains a sequence number, contained in RsslUpdateMsg::seqNum.

RSSL_UPMF_HAS_CONF_INFO 

(0x020) Indicates that this RsslUpdateMsg contains conflation information. This information is contained in RsslUpdateMsg::conflationCount and RsslUpdateMsg::conflationTime.

RSSL_UPMF_DO_NOT_CACHE 

(0x040) Indicates that this RsslUpdateMsg is transient and that this message's payload information should not be cached

RSSL_UPMF_DO_NOT_CONFLATE 

(0x080) Indicates that this RsslUpdateMsg's payload information should not be conflated or aggregated.

RSSL_UPMF_DO_NOT_RIPPLE 

(0x100) Indicates that rippling should not be applied to this RsslUpdateMsg's contents.

RSSL_UPMF_HAS_POST_USER_INFO 

(0x200) This RsslUpdateMsg has been posted by the user identified in RsslUpdateMsg::postUserInfo.

RSSL_UPMF_DISCARDABLE 

(0x400) Indicates that this RsslUpdateMsg can be discarded. This flag is commonly used with options that have no open interest.
