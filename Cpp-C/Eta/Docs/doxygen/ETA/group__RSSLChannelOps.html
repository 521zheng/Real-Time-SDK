RSSL Client and Server Channel Operations
RSSL Channel Operations

These functions are used by both the Client and Server to finalize the rsslChannel initialization, read from the channel, write to the channel, and request buffers from the shared pool buffers.

RSSL_API RsslRet rsslInitChannel ( RsslChannel *  chnl,
RsslInProgInfo *  inProg,
RsslError *  error 
)

Typical use:

  1. Connect using rsslConnect()
  2. While Channel state is RSSL_CH_STATE_INITIALIZING and the channels socketId detects data to read, call rsslInitChannel()
Note:
This is not necessary for blocking channels, which will return in the RSSL_CH_STATE_ACTIVE state after the rsslConnect() call.
Parameters:
chnlRSSL Channel to continue initialization on
inProgInProg Info for compatibility mode reconnections
errorRSSL Error, to be populated in event of an error
Returns:
RsslRet RSSL return value or RsslReturnCodes value

RsslReturnCodes, RsslInProgInfo
