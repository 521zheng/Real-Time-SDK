RSSL_API RsslRet rsslEncodeNonRWFDataTypeComplete ( RsslEncodeIterator *  pIter,
RsslBuffer *  pBuffer,
RsslBool  success 
)
  1. Use your rssl Init function (e.g. rsslEncodeMsgInit, rsslEncodeMapEntryInit) for the container you intend to nest non-RWF data into

Call rsslEncodeNonRWFDataTypeInit with the iterator being used for encoding and a RsslBuffer structure.

  1. rsslEncodeNonRWFDataTypeInit will populate the pBuffer->data with the address where encoding should begin and the pBuffer->length with the amount of space left to encode into
    1. When NonRWF encoding is complete, set pBuffer->length to actual number of bytes encoded and pass pIter and pBuffer to rsslEncodeNonRWFDataTypeComplete
  2. Finish RWF container encoding (e.g. message, map entry, etc).
    Note: User should ensure not to encode or copy more than pBuffer->length indicates is available
    Parameters:
    pIterRsslEncodeIterator used for encoding container
    pBufferRsslBuffer obtained from rsslEncodeNonRWFDataTypeInit. pBuffer->length should be set to number of bytes put into pBuffer->data. pBuffer->data must remain unchanged.
    successif RSSL_TRUE, encoding was successful and RSSL will complete transfer of data. If RSSL_FALSE, RSSL will roll back to last successful encoding prior to Non-RWF Init call.
