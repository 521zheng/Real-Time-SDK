RsslGenericMsg Struct Reference
RsslGenericMsg Structure, Initializers, and Flag values

The RsslGenericMsg allows applications to bidirectionally send messages without any implied interaction semantics (e.g. no Request/Response paradigm). More...

#include <rsslGenericMsg.h>

Data Fields

RsslBuffer extendedHeader
 Extended header attributes, used for per-domain defined attributes that are not used when determining stream uniqueness. Presence is indicated by RsslGenericFlags::RSSL_GNMF_HAS_EXTENDED_HEADER.
RsslUInt16 flags
 Flag values used to indicate optional member presence and/or stream behavior. The available options are defined by values present in RsslGenericFlags.
RsslMsgBase msgBase
 Common message header members (streamId, domainType, msgKey, etc.). The msgKey is optional.
RsslUInt16 partNum
 Part Number is a 15-bit value typically used with multi-part generic messages. Value should start with 0 to indicate the initial part, each subsequent part in a multi-part generic message should increment the previous partNum by 1.
RsslBuffer permData
 Contains authorization information for this stream. Presence is indicated by RsslGenericFlags::RSSL_GNMF_HAS_PERM_DATA.
RsslMsgKey reqMsgKey
 Optional Key providing unique identifier information for an item stream. If present, this contains the original request message key and is used to uniquely identify a stream.
RsslUInt32 secondarySeqNum
 Secondary Sequence Number often used as an acknowledgment sequence number. Presence is indicated by RsslGenericFlags::RSSL_GNMF_HAS_SECONDARY_SEQ_NUM.
RsslUInt32 seqNum
 Sequence number intended to help with temporal ordering. Typically, this will be incremented with every message, but may have gaps depending on the sequencing algorithm being used. Presence is indicated by RsslGenericFlags::RSSL_GNMF_HAS_SEQ_NUM.
