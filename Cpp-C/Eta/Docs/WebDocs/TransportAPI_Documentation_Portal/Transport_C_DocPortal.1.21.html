OMM Consumer Watchlist
RsslReactor features a watchlist that provides extra functionality for OMM consumer applications. When enabled, the reactor performs various recovery and aggregation behaviors that you would otherwise need to implement yourself. These behaviors include:
•
Item Recovery: When a consumer application sets SingleOpen in its login request, the RsslReactor will automatically respond to failure conditions (such as reconnections or unavailable services), and automatically recover items requested by the application.
•
Item Aggregation: To prevent providers from sending redundant data, when a consumer application requests multiple streams which share item information, the RsslReactor will aggregate them into a single stream. If these streams request features such as views, or pausing or resuming, these features will also be aggregated.
•
Group & Service Status Fanout: The RsslReactor maintains a directory stream to receive service updates. As the RsslReactor receives group status messages or service status messages, the RsslReactor forwards the associated status to all affected streams via RsslStatusMsgs.
•
QoS Range Matching: The RsslReactor will accept and aggregate item requests that specify a range of RsslQos, or requests that do not specify an RsslQos. These requests will be matched against the QoS available from the providing service, and the best matching QoS used.
•
Support for Enhanced Symbol List Behaviors: The RsslReactor supports requesting data streams when requesting a Symbol List item.
•
Support for Batch Requests: The RsslReactor will accept batch requests, even if the provider does not natively support them.
