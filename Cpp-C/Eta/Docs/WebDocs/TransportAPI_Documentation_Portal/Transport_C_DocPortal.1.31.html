Consumer
rsslConsumer is a single-threaded, client application that consumes or posts content between an OMM consumer and OMM provider. rsslConsumer:
•
Initializes the Transport API transport and connects the client.
•
Sends a login request, a source directory request, a dictionary request, and one or more item request messages (snapshot or streaming) to a provider
•
Processes the responses and displays them in the console.
rsslConsumer supports consuming Level I Market Price, Level II Market By Order, Level II Market By Price, and Yield Curve information as normal or private streams.

NIProvider
rsslNIProvider is a single-threaded, client application that non-interactively provides Level I Market Price and Level 2 Market By Order data to an Advanced Data Hub (ADH). rsslNIProvider:
•
Initializes the Transport API transport and connect the client.
•
Loads the dictionary from a file. If the dictionary cannot be loaded, the application exits.
•
Sends a login request like a consumer to the ADH and processes dictionary requests like a provider.
•
Publishes source directory refresh messages, market price, and/or market-by-order refresh/update messages to the ADH (without any request for them).
You can use reliable multicast to communicate between this application and any ADH on the network. Using multicast, the non-interactive provider can send one message to all ADH's on the network instead of having to fan-out messages to each ADH TCP/IP connection.

Provider
rsslProvider is a single-threaded, server application that interactively provides Level I Market Price, Level II Market-By-Order, Level II Market-By-Price, and Symbol List data to one or more consumers. rsslProvider:
•
Initializes the Transport API transport and binds the server.
•
Loads dictionary information from the RDMFieldDictionary and enumtype.def files.
•
Processes login, source directory, dictionary, market price, market-by-order, market-by-price, and symbol list requests from consumers.
•
Sends appropriate responses.
rsslProvider sends Level II Market-By-Price refresh messages as multi-part messages. When sending multi-part refresh messages, rsslProvider includes an update message between each part.
