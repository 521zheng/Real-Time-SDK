What are Value Added Components?
Value Added Components are offered alongside the native Transport API to maximize the user experience by allowing developers to more easily and rapidly develop Transport API applications. Value Added Components provide alternative and simpler entry points for applications to leverage the OMM-based Transport API. As the application developer, you can use Value Added Components to choose and customize the balance between ultra-high-performance raw access and ease-of-use feature functionality. Itâ€™s up to you to decide whether to leverage these components: the use of Value Added Components is optional.
In addition, the application can utilize some or all of the Value Added Components. Value Added Components are written to the Transport API and are designed to work alongside it. As a result and also to provide simple migration and encourage consistency of use, the Value Added Component interfaces have a similar look and feel to the Transport API interfaces.
