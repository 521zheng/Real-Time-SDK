rsslGetBuffer Return Values
The following table defines return and error code values that can occur while using rsslGetBuffer.
 
Table 46: rsslGetBuffer Return Values  
RETURN CODE
DESCRIPTION
Valid buffer returned
Success Case
An RsslBuffer is returned to the user. The RsslBuffer.length indicates the number of bytes available to populate, and RsslBuffer.data provides a starting location for population.
NULL buffer returned
Error Code: RSSL_RET_BUFFER_NO_BUFFERS
NULL is returned to the user. This value indicates that there are no buffers available to the user. See RsslError content for more details.
This typically occurs because all available buffers are queued and pending flushing to the connection. The application can use rsslFlush to attempt releasing buffers back to the pool (refer to Section 10.10.2). Additionally, the rsslIoctl function can be used to increase the number of guaranteedOutputBuffers (refer to Section 10.14).
NULL buffer returned
Error Code: RSSL_RET_FAILURE
NULL is returned to the user. This value indicates that some type of general failure has occurred. The RsslChannel should be closed, refer to Section 10.13. See RsslError content for more details.
NULL buffer returned
Error Code: RSSL_RET_INIT_NOT_INITIALIZED
Indicates that the RSSL Transport has not been initialized. See the RsslError content for more details. For information on initializing, refer to Section 10.2.
