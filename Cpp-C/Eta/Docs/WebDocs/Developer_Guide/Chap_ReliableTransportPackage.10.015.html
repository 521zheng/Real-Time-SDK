RSSL Initialization and Uninitialization Functions
The following table provides additional information about the RSSL Transport functions used for initializing and uninitializing.
 
Table 18: RSSL Initialization and Uninitialization Functions  
function
Description
rsslInitialize
The first RSSL Transport function that an application should call. This creates and initializes internal memory and structures, as well as performing any boot strapping for underlying dependencies. The rsslInitialize function also allows the user to specify the locking model they want applied to the RSSL Transport. For more information, refer to Section 10.2.3.
rsslUninitialize
The last RSSL Transport function that an application should call. This uninitializes internal data structures and deletes any allocated memory.
