Market By Price Refresh Message
A Market By Price refresh message is encoded using RefreshMsg and sent by OMM interactive provider and OMM non-interactive provider applications.
A Market By Price refresh may be sent in multiple parts. Both update and status messages can be delivered between parts of a refresh message, regardless of streaming or non-streaming request.
 
Note: The provider should send Name and ServiceName only in the first Refresh response message. However if EMA is configured with MsgKeyInUpdates set to true, then Name and ServiceName must be provided in each Refresh response message.
